# src/lib/hooks/usePushSubscription.ts

React-хук Web Push подписки. Возвращает { isSupported, isSubscribed, permission, isLoading, subscribe, unsubscribe }. isSupported = window.isSecureContext && 'serviceWorker' in navigator && 'PushManager' in window && 'Notification' in window (БЕЗ зависимости от env, чтобы не блокировать UI). В useEffect синхронизирует permission и текущую подписку через navigator.serviceWorker.ready + pushManager.getSubscription. subscribe(): requestPermission → fetch('/api/notifications/vapid-public-key') → pushManager.subscribe({userVisibleOnly:true, applicationServerKey:urlBase64ToUint8Array(publicKey)}) → POST /api/notifications/subscribe. unsubscribe(): pushManager.getSubscription → sub.unsubscribe() + DELETE /api/notifications/subscribe. VAPID key больше НЕ инлайнится в клиентский бандл — приходит с бэкенда через /api/notifications/vapid-public-key, благодаря чему смена ключа не требует пересборки фронта, а пустая env-переменная не ломает isSupported.
